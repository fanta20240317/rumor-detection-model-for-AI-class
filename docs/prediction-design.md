# Prediction Design

Terminal and web prediction use `RumorPredictionService`.

The shared service loads the same `RumorDetectionPipeline` used by evaluation,
then attaches optional school-LLM evidence after the local model has already
produced its label, probability, threshold, and structured evidence.

The JSON includes label, probability, confidence, threshold, baseline fields,
`evidence`, local `explanation`, and `llm_evidence`.

`evidence` contains keyword contributions, retrieved training cases, retrieval
statistics, claim-structure features, decision factors, and probability guard
details.

`llm_evidence` is enabled by default. It is generated by the configured school
LLM from the local model output and evidence only; it must not change the final
label or introduce outside facts.

学校提供的大语言模型接口作为解释润色模块。分类标签由本地可复现模型输出，
LLM 不参与标签决策；系统将原始推文、预测标签、置信度、模型证据词和
RAG 相似样本传入学校模型，由其生成自然语言判断依据。

If the school API is not configured or the request fails, prediction still
returns local model evidence and sets `llm_evidence.available` to `false`.

The default school-LLM endpoint is the SJTU OpenAI-compatible API at
`https://models.sjtu.edu.cn/api/v1`, and the default model is `deepseek-chat`.
The only required environment variable for LLM evidence is
`SCHOOL_LLM_API_KEY`. `SCHOOL_LLM_MODEL`, `SCHOOL_LLM_BASE_URL`, and
`SCHOOL_LLM_API_URL` are optional overrides.

`make predict` uses the default model and train evidence paths with an
overridable `TEXT` variable. `python predict.py --no-llm ...` disables LLM
evidence for one terminal run.

`make web` starts the local browser UI, which calls the same prediction service
and displays the same model evidence plus LLM evidence.

Missing model artifacts or evidence corpus paths should fail clearly instead of
silently falling back to another model.
